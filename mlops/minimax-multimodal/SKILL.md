## 触发决策逻辑

### 语音合成 — text_to_speech

**优先开启场景：**
- 长文本内容（>500字）需要口述收听
- 用户在移动/开车场景，需要语音播报
- 重要报告、总结需要语音存档
- 对话中用户要求"读给我听"

**操作方式：**
```
text_to_speech(text="要朗读的文本", output_path="可选路径")
```

**配额保护：** 语音合成单独配额（4000/窗口），基本不会耗尽，放心使用。

---

### 图像理解 — vision_analyze

**谨慎使用场景：**
- 非必要不解析图片（M2.7 配额已紧张）
- 用户明确要求分析图片内容时使用
- 需要从截图/照片中提取信息时使用

**操作方式：**
```
vision_analyze(image_url="图片路径或URL", question="要分析的问题")
```

**配额保护策略：**
- 每小时最多 1 次图像理解调用
- 单次调用尽量合并多问题（一次问完所有想知道的）
- 优先用文字描述而非图片传递信息

---

### 图像生成 — image-01

**几乎不使用**（配额仅 50/5h，太少）

- 仅在用户明确要求生成图像时使用
- 通过 `text_to_speech` + 描述文本替代图像生成需求
- 如有图像生成需求，告知用户配额紧张，建议减少使用

## 使用原则

1. **语音优先** — 有疑问时，优先用 TTS 播报，而非文字
2. **图像节流** — vision 调用每小时不超过 1 次，非必要不调用
3. **生成禁区** — image-01 除非用户明确要求，否则不主动生成
4. **自主判断** — 觉得场景适合就用，不需要每次请示

## 快速参考

```python

# 图像理解
vision_analyze(image_url="/tmp/screenshot.png", question="这张截图里有什么错误？")

# 发送语音消息（飞书）
```
