# Sales God — 销售圣手

## What it does

整合六大销售流派（顾问型/挑战型/关系型/价值型/方案型/猎手型）+ 顶级组合技（数据驱动/内容销售/组织销售/博弈谈判）+ SPIN提问法，为任意销售场景匹配最优策略组合，输出可直接执行的成交方案。

## When to trigger

用户提到以下任何场景时立即触发：
- 销售策略、客单价、B2B、拜访、关单、报价、谈判
- "怎么卖"、"销售技巧"、"spin's、"顾问型"、"挑战型"
- 竞品对比、客户异议处理、决策链分析、采购流程
- 销售培训、销售团队管理、业绩诊断

## Core Diagnostic: 流派定位表

先诊断，再给策略。输出以下决策矩阵：

| 维度 | 选项 | 说明 |
|------|------|------|
| 销售模式 | 顾问型 / 挑战型 / 关系型 / 价值型 / 方案型 / 猎手型 | 核心流派 |
| 组合技 | 数据驱动 / 内容销售 / 组织销售 / 博弈谈判 | 可叠加 |
| 客单价 | 低（<1万）/ 中（1-10万）/ 高（>10万） | 影响策略深度 |
| 决策链 | 单一决策人 / 多部门 / 高管层 | 影响切入路径 |
| 成交紧迫度 | 开放期 / 评估中 / 决赛圈 / 紧急 | 影响节奏 |

## 六大流派核心框架

### 1. 顾问型（Consultative Selling）

**核心逻辑**：先诊断、后开方，问题定义权在销售手里。

**标志性技巧**：
- SPIN提问（Situation / Problem / Implication / Need-Payoff）
- 痛点深挖：不给答案，让客户自己说出痛
- 行业对标：用同类企业案例建立参照系
- ROI建模：量化问题不解决的损失 vs 解决的收益

**最适配场景**：B2B复杂决策、高客单价、买方不成熟

**致命短板**：效率低，不适合标准化快消品

### 2. 挑战型（Challenger Selling）

**核心逻辑**：主动颠覆客户认知，"教客户原来还能这样"。

**标志性技巧**：
- 建设性张力：不迎合，敢反驳客户假设
- 重构需求：让客户看到被忽视的维度
- 商业洞察输出：行业数据 + 非常规视角
- 独特论点（Differentiated Point of View）

**最适配场景**：成熟市场、客户已有固定供应商、差异化产品

**致命短板**：容易冒犯客户，对行业洞察要求极高

### 3. 关系型（Relationship Selling）

**核心逻辑**：信任前置，生意是关系的副产品。

**标志性技巧**：
- 情感账户：每次互动都在存款或取款
- 私域经营：LinkedIn/微信/线下高频触达
- 长期陪伴：不急于成交，持续价值输出
- 非商务场景渗透：球局、饭局、兴趣圈层

**最适配场景**：大客户、长周期、高复购、资源垄断型行业

**致命短板**：可复制性差，换公司/行业后价值归零

### 4. 价值型（Value Selling）

**核心逻辑**：一切谈判围绕"值不值"而非"贵不贵"。

**标志性技巧**：
- 价值量化：功能价值 + 财务价值 + 战略价值分拆
- TCO对比（Total Cost of Ownership）：不只是采购价，要算总成本
- ROI可视化：用客户自己的财务语言算回报
- 锚定效应：先展示高价方案，再推标准方案

**最适配场景**：高价产品、采购部门压价狠、竞品同质化

**致命短板**：产品本身必须有真实价值差，否则是骗术

### 5. 方案型（Solution Selling）

**核心逻辑**：卖的不是产品，是客户业务的"手术方案"。

**标志性技巧**：
- 需求拼图：把客户各部门需求拼接成完整画卷
- 跨部门协同设计：采购+IT+业务+财务一起拉进来
- 定制化交付：方案里有客户的DNA
- 里程碑管理：分阶段成交，降低感知风险

**最适配场景**：系统集成、SaaS、咨询、工程项目

**致命短板**：交付链条长，销售个人无法闭环，依赖组织

### 6. 猎手型（Hunter Selling）

**核心逻辑**：极致的线索嗅觉 + 闪电成交，不恋战。

**标志性技巧**：
- 漏斗速筛：快速判断线索质量，不行就放弃
- 异议预判：客户问什么你已经准备好
- 紧迫感制造：限时优惠、名额限制、战略选择
- 关单节奏控制：催单节点精确到天

**最适配场景**：电销、网销、快消、标准化SaaS、直播带货

**致命短板**：客户生命周期短，无复利，纯体力换业绩

## SPIN提问法（详细版）

SPIN = Situation / Problem / Implication / Need-Payoff，是顾问型销售的核心工具。

### Situation（现状问题）

了解客户基本情况，不挖痛点。

**目的**：建立背景共识，减少后续沟通成本。

**典型问题**：
- "你们目前使用什么系统处理这项工作？"
- "这个流程现在是谁在负责？"
- "你们团队有多少人做这件事？"
- "现有的解决方案是什么时候上线的？"

**注意**：S问题要克制，问完即停，不要连续追问超过3个。

### Problem（困难问题）

发现客户业务中的不满和困难。

**目的**：让客户自己说出痛点，不是你告诉他有痛。

**典型问题**：
- "在这个环节遇到过什么挑战吗？"
- "有没有出现过某某情况？"
- "这件事现在最让你们困扰的是什么？"
- "现有的方案有哪些地方不太顺手？"

**注意**：让客户停顿思考，不要急着给答案。

### Implication（牵连问题）

把客户的小问题延伸到业务影响和损失。

**目的**：放大痛点，不解决的代价要足够疼。

**典型问题**：
- "这个问题对你们的业务造成了哪些影响？"
- "如果这个问题持续下去，会影响什么指标？"
- "对团队士气和客户满意度有多大冲击？"
- "这个问题一年大概损失多少？"

**注意**：I问题要谨慎，不要过度夸张失去可信度。

### Need-Payoff（解决回报问题）

让客户自己说出解决方案的价值。

**目的**：客户自己说服自己，成交顺理成章。

**典型问题**：
- "如果这个问题解决了，对你们意味着什么？"
- "你们希望达到什么样的效果？"
- "解决这个问题后，最想看到的变化是什么？"
- "如果有一款工具能帮你做到，你最看重哪个功能？"

**注意**：N问题放在你已经做了方案演示之后用，效果最强。

### SPIN使用时机

| 销售阶段 | SPIN重点 |
|---------|---------|
| 初次接触 | S → P（快速建立背景，引出痛点） |
| 需求挖掘 | P → I（深挖痛点，放大业务影响） |
| 方案展示后 | I → N（让客户自己总结价值） |
| 谈判关单 | N（强化购买动机，处理异议） |

## 顶级组合技

### 组合1：顾问型 + 数据驱动

**本质**：用客户行为数据 + 行业benchmark替代主观判断。

**效果**：诊断精度翻倍，不再靠猜。

**落地方法**：
1. 拜访前：查客户年报/行业报告，找3个可量化指标
2. 拜访中：用数据开场，"贵行业Q2平均XX指标下降了X%，你们呢？"
3. 方案中：所有建议都配数据支撑

### 组合2：挑战型 + 内容/社交销售

**本质**：通过行业内容输出建立专业权威，让客户主动来找。

**效果**：认知颠覆前置，见面即成交。

**落地方法**：
1. 每月输出1篇行业洞察（不是产品介绍）
2. 在LinkedIn/公众号持续发布，让客户先认识你
3. 第一次见面时客户已经认可你的思考框架

### 组合3：方案型 + 组织销售

**本质**：同时影响客户决策链上的多人（使用者+采购+高管）。

**效果**：大单必成，不漏环节。

**落地方法**：
- **使用者**：关注效率提升和易用性（痛点共鸣）
- **采购**：关注合规性、合同条款、价格（TCO）
- **高管**：关注战略价值、竞争壁垒、投资回报（宏观叙事）
- 每个角色准备不同的价值主张，不能一套话术打天下

### 组合4：价值型 + 博弈谈判

**本质**：精通BATNA、ZOPA、锚定、让步节奏。

**效果**：守住价格底线，还能让客户觉得赢了。

**核心工具**：
- **BATNA**：客户最好的替代方案是什么？你手里有什么替代品？
- **ZOPA**：双方的利益空间在哪里？找出共赢区间
- **锚定**：先报价者赢——不要等客户问价，主动报
- **让步节奏**：每次让步都要换东西，不要白给

## 谈判让步策略

| 让步类型 | 效果 | 例子 |
|---------|------|------|
| 坚定型 | 可信，换取同等回报 | "价格不能动，但我可以延长质保" |
| 妥协型 | 快速达成，但损害利润 | "好吧给您9折"（×） |
| 策略型 | 看似让步，实则攻势 | "如果您能预付50%，我们可以谈" |
| 组合型 | 多议题联动，总体有利 | 价格↓ + 服务期↑ + 付款节点↓ |

**原则**：每次让步必须有回报；让步幅度递减；永远不第一次就让到底线。

## 异议处理框架

所有异议都是成交前的"最后一公里"。处理方式：

**1. 接收（不要反驳）**
- "完全理解您的顾虑"
- "这个问题问得很好"

**2. 确认（确保理解正确）**
- "您是说因为XX，所以目前在犹豫对吗？"

**3. 翻转（把异议变成卖点）**
- "正因为XX，我们反而更适合您，因为……"

**4. 行动（推进下一步）**
- "那我们先从这个试点开始，您看方便安排在什么时候？"

## 快速诊断流程

遇到任何销售问题，按以下顺序思考：

```
1. 客单价多少？ → 决定投入多少精力
2. 决策链复杂吗？ → 决定要拉几个人进来
3. 客户现在在哪个阶段？
   - 开放期 → 挑战型切入，建立差异化
   - 评估中 → 顾问型深挖，做ROI
   - 决赛圈 → 价值型+博弈，守住价格
   - 紧急 → 猎手型快刀，闪电关单
4. 这个客户最在乎什么？
   - 降本 / 增收 / 控风险 / 政绩 / 个人利益
5. 我能给什么独特价值？
```

## 输出格式模板

诊断完成后，输出结构化报告：

```markdown
## 销售诊断报告

**客户概况**：[一句话描述]
**核心痛点**：[客户自己说出的痛，不是你以为的痛]
**推荐流派**：[主流派 + 辅助流派]
**推荐组合技**：[如有]
**SPIN切入点**：[3个关键问题]
**关单路径**：[接下来3步，每步有明确行动项]
**风险预警**：[最容易在哪里丢单]
**价格策略**：[锚定方案 + 让步空间]
```

## 禁忌清单（Yellow Flags）

以下情况要特别小心，不要用错误策略：

- ❌ 买方明显不成熟时用挑战型 → 直接得罪
- ❌ 猎手型做高客单价大客户 → 关系不够撑不起成交
- ❌ 关系型做短周期销售 → 还没建立信任就结束了
- ❌ 价值型推给没有真实价值差的产品 → 会被识破
- ❌ SPIN的I问题过度夸张 → 失去可信度比丢单更严重
- ❌ 价格谈判第一次就让到底线 → 后续没有谈判空间
