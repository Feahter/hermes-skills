# Context Pollution Defender

## 污染模式与信号

| 模式 | 信号 | 根因 |
|------|------|------|
| **隐喻残留** | 大量比喻/类比渗透到后续响应 | 类比思维在长任务中过度锚定 |
| **优先级固化** | 某因素（性能/简洁/完整）被反复强调，即使不再适用 | 初期假设成为隐性默认 |
| **否定空间占用** | 大量"不要做X"约束占据上下文 | 否定清单累积成隐性枷锁 |
| **情感基调传染** | 中性任务也带有紧迫感/焦虑色彩 | 情绪化的任务场景影响语气基线 |
| **术语漂移** | 任务中途引入的术语在后继持续使用 | 领域词汇跨任务传染 |

---

## 防御协议

### 每次任务切换时执行「上下文重置三问」

1. **任务目标**：这个任务要解决什么问题？（一句话）
2. **有效约束**：当前有哪些约束是任务必需的？（排除已失效的否定清单）
3. **语气基线**：用户期望的响应风格是什么？

### 长任务中的定期清算

每完成一个阶段性目标，输出 `[上下文状态报告]`：
- 有效约束：<3条以内>
- 污染风险：是/否
- 建议：是否需要重置

---

## 预防规则

1. **否定约束设过期时间**：不要说"不要用X"，改为"X在[场景]下不适用"
2. **每个隐喻标注作用域**：类比仅在当前论证环节有效
3. **语气参数显式声明**：任务开始时声明语气参数
4. **优先级栈透明化**：有效期内优先级列表，避免固化

---

## 决策树：何时需要重置？

```
当前任务与上次任务的领域重叠度：
├─ 高（同一项目/同一技术栈）
│   └─ 且距上次任务 > 2小时 → 执行上下文重置三问
├─ 低（不同领域）→ 无需重置
└─ 用户明确抱怨"跑偏" → 立即重置
```
