# 天才思维方法论

*三层递归操作系统——不是更聪明，而是运行着自我迭代的元认知系统*

---

## 核心公式

```
天才 = 系统A（直觉） × 系统B（逻辑） × 递归深度
```

**为什么有效：** 普通人的双系统是串行的——先用A想，再用B检查。天才让两系统在张力中同时运转，制造认知摩擦，从中诞生新想法。

---

## 输入输出契约

**输入：** 一个难题 / 一个显然的结论 / 一个卡住的思维困境
**输出：** 三选一——反例清单 / 三种表征视角 / 元层质疑

**触发检查（用前自评）：**
```
我需要：□反例  □新视角  □跳出框架
当前卡在哪：________________
```

---

## 执行流程

### STEP 1: 矛盾探测器（系统A → B 切换）

**动作：** 识别"显然"的结论，强制找反例

```
原始结论：___________________
[反例] 这个结论在 ___ 条件下失效？
[张力] 成立的那部分说明了什么？
```

**为什么有效：** 系统A瞬间抓住模式（快），系统B追问"失效条件"（慢）——两者的摩擦点就是新认知的出生地。

### STEP 2: 表征转换器（三种语言切换）

**动作：** 同一问题用三种不同表征重新表达

```
原始表征：___（语言A）
[表征B] 如果用数学/逻辑怎么表达？
[表征C] 如果用视觉/空间怎么表达？
[表征D] 如果用一个故事/隐喻怎么表达？
```

**收敛检查：** 其中一种表征是否暴露了其他两种都没触及的盲区？

### STEP 3: 元认知钩子（递归出口）

**动作：** 跳出当前思考，观察思考过程本身

```
[对象层] 我在思考什么？ → 答案：___
[过程层] 我为什么这样思考？ → 答案：___
[元层]  我用的这个分类标准本身对吗？
```

**收敛检查：** 元层的质疑是否暴露了前两层的前提错误？

---

## 四组对立统一（工具箱，非流程）

| 张力 | 操作 | 典型提问 |
|------|------|---------|
| 分解 ↔ 综合 | 分解时预判重组，综合时持续拆解 | "我拆完后能重新组装回去吗？" |
| 抽象 ↔ 具体 | 抽象概念同时调用体感具象 | "说到万有引力，脑中有没有苹果落地？" |
| 怀疑 ↔ 确信 | 内容怀疑，方法暂时确信 | "这个结论的反例是什么？→方法本身呢？" |
| 自我 ↔ 无我 | 剥离个人偏好，模拟系统视角 | "如果我是对手/自然/代码，会怎么看？" |

---

## 最小可行内核（快速启动）

如果时间紧迫，只用三个组件：

```
矛盾探测器 → 表征转换器 → 元认知钩子
（找反例）   （换语言）     （跳出来）
```

---

## 负向约束（为什么这样设计）

```
硬负向：
- 不要在系统A里停留超过30秒——直觉出现后立即切换B
- 不要同时用四组张力——选当前最卡的那一对
- 不要跳过表征直接到结论——必须有"换语言"这一步

软负向：
- 尽量控制案例数量——每个机制最多一个就够了
- 尽量不用"显然"这个词——它是大脑偷懒的信号
```

**为什么这样约束？** 天才思维的核心是"摩擦"，而不是"加速"。给系统A和B足够的时间碰撞，比快速产出更重要。

---

## 验证清单（输出后必做）

```
Before finalizing, verify:
☐ 矛盾探测器：是否真的找到了反例，还是只是在正例里打转？
☐ 表征转换器：是否切换了语言，还是只是换了同义表述？
☐ 元认知钩子：元层质疑是否比对象层/过程层更深一层？
☐ 自评分数：整体思考质量 ≥ 4/5，否则从 STEP 1 重来
```

## 执行口诀

```
直觉来 → 找反例
单一视角 → 切三种
沉浸思考 → 跳出来
理所当然 → 问负空间
```

---

## 关键原理（解释 why）

**为什么是三层，不是两层？**
两层（直觉+逻辑）只能校准答案。三层加入"元认知"，可以校准问题本身——这是质的飞跃。

**为什么"显然"是危险信号？**
"显然"是系统A单独决策的产物，没有经过系统B的检验。每发现一个"显然"，就发现了一个认知盲点。

**为什么表征转换能发现盲区？**
每种语言天然会遗漏某些信息。三种语言的交集才是真正的理解，三种语言的差集才是新的发现。
