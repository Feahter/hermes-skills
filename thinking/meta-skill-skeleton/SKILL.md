# Meta-Skill Skeleton

## 一、SKILL.md 完整字段清单

按影响力排序：

| 字段 | 为什么关键 | 新手错误 |
|------|----------|---------|
| `description` | 触发匹配度，决定skill是否被加载 | 只写"用于X"，不写触发关键词 |
| **`反例/负面模式`** | 告诉AI何时skill失效 | 只写正面，不写"不用于Y" |
| **`输入输出契约`** | 明确输入格式和输出格式 | 模糊导致散文输出 |
| **`依赖声明`** | 前置skill/工具 | 隐性依赖导致组合失败 |
| `triggers` | 自然语言触发词 | 缺少同义词覆盖 |
| `Pitfalls` | 常见错误和禁忌 | 缺失导致重复踩坑 |
| **`认知负荷预算`** | 何时需要拆分子agent | 忽视导致上下文溢出 |
| **`版本兼容性`** | 工具版本差异 | 过时产生破坏性建议 |
| `Verification` | 如何验证skill执行正确 | 缺失导致质量不稳定 |

## 二、Emergent Behavior（涌现行为）记录方法

**5种高价值用法：**

1. **反脆弱性测试**：记录"skill A + skill B 意外好" → 反推"skill A + skill C 是否意外坏" → 构建组合安全矩阵
2. **能力边界探测**：涌现暴露你以为AI不会但实际会的能力 → 更新技能雷达图
3. **压缩表示**：高频涌现组合固化为新skill时，原记录作训练数据
4. **故障归因**：系统异常时，检查是否是历史涌现组合被意外触发
5. **创新发现**：当skill A的能力被skill B补足时，发现全新能力组合

**记录格式（在skill的术层）：**
```
### 涌现行为
- [日期] skillA + skillB → 效果描述
- 风险：skillA + skillC → 潜在问题
```

## 三、Skill 创作 Checklist

- [ ] `description` 包含触发词（P=1.0 R=1.0 测试通过）
- [ ] 有明确的"不用于Y"反例
- [ ] 输入输出格式明确（自然语言模板或JSON schema）
- [ ] 依赖的skill/工具已声明
- [ ] 有认知负荷预算（何时拆分子agent）
- [ ] 有版本兼容性说明
- [ ] 有Pitfalls/禁忌
- [ ] 有Verification验证步骤
- [ ] 已在道/法/术/器四层中找到定位
