# 小说创作SOP - 严格版 v3.0
## 基于「种子→萌芽→枝干→枝叶」模型

## 核心理念

小说创作如同种植一棵故事之树：
```
种子（概念）→ 萌芽（大纲）→ 枝干（分镜）→ 枝叶（内容）
```

| 层级 | 产出 | 核心任务 |
|------|------|---------|
| **种子** | 核心概念卡 | 确定书名、类型、金手指、爽点、主题 |
| **萌芽** | 主线剧情树 | 规划主线、冲突、人物弧光、转折点 |
| **枝干** | 卷纲+章纲 | 设计章节结构、场景安排 |
| **枝叶** | 正文+摘要 | 填充具体内容、章节写作 |

---

## 触发条件

当用户要求以下操作时使用此Skill：
- 继续小说创作
- 创作新章节
- 从零开始规划新小说
- 修复创作流程问题
- 批量生成章节

---

## 📋 核心原则

| 原则 | 说明 |
|------|------|
| **每步必保存** | 完成任何步骤立即保存进度 |
| **错误不过夜** | 出错立即记录，继续下一章 |
| **层级清晰** | 每个创作阶段有明确产出物 |
| **锁文件超时** | 超过30分钟自动清理 |
| **幂等执行** | 同一章节可重复执行 |
| **Phase 3无需确认** | 进入创作阶段后禁止向用户确认，直到完成 |

## 偏好记忆系统 v5.0融合chinese-novelist

> 来源：chinese-novelist-skill（GitHub, MIT License）

### 存储文件
`{小说目录}/user-preferences.json`（项目根目录，首次使用后自动创建）

### 数据结构
```json
{
  "version": 1,
  "updatedAt": "2026-04-21",
  "preferences": {
    "favoriteGenres": [],
    "preferredProtagonist": "",
    "preferredPerspective": "",
    "preferredTone": "",
    "typicalChapterCount": null,
    "styleReferences": [],
    "dislikes": [],
    "creationHistory": []
  }
}
```

### 偏好更新规则
| 时机 | 行为 |
|------|------|
| 每完成一层问答 | 静默将本层回答同步到偏好文件 |
| 用户说"记住我的偏好" | 保存当前所有配置到偏好 |
| 用户说"忘记XX偏好" | 清除指定维度偏好 |
| 用户说"重置偏好" | 清空所有偏好数据 |
| 一部长篇创作完成 | 将作品信息追加到 `creationHistory` |

### 偏好如何影响交互
1. **启动欢迎语**：有偏好时显示"欢迎回来！"+ 个性化提示
2. **选项排序**：将 `favoriteGenres` 匹配项排前面，⭐标记常用项
3. **随机生成**：优先从偏好范围内随机选取，保持一致性
4. **中断续写**：Phase 0 读取 JSON 检测未完成项目，询问"继续上次的《XXX》？"

## 中断续写机制 v5.0融合chinese-novelist

### 检测逻辑
每次启动时读取 `{小说目录}/progress.json`：
- 存在 `status: "in_progress"` 的章节 → 从该章节继续
- 存在 `status: "failed"` 的章节 → 从第一个 failed 章节开始（最多3轮重写）
- 所有章节 `status: "completed"` → 询问是否开启新卷

### 偏好文件损坏处理
JSON 解析失败时：忽略偏好，使用默认值，并在后台修复文件。

## 三种写作模式 v5.0融合chinese-novelist

> Phase 2.5 写作模式选择：规划确认后，用户选择模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **串行（serial）** | 主 Agent 逐章写，稳定可靠 | 默认推荐 |
| **子Agent并行（subagent-parallel）** | 多个子 Agent 分批并行写 | 追求速度 |
| **Agent Teams（agent-teams）** | Claude Code 多 Agent 协作 | 大型长篇 |

### 串行模式执行规则
> ⚠️ **Phase 3 核心约束**：一旦进入创作阶段，**禁止使用 AskUserQuestion/clarify 工具向用户确认**，必须逐章执行直到全部完成。本章完成后立即读取 progress.json 认领下一章，不要停下来。

### 子Agent并行模式
主 Agent 将章节分成不重叠的批次（每批 5-8 章），每个批次派生一个子 Agent。批次内串行，批次间并行。并发安全：每个子 Agent 负责不重叠的章节批次，通过 `progress.json` 状态避免写入冲突。

### Agent Teams 模式
使用 TeamCreate 创建写作团队，派生 3-5 个团队成员，通过 TaskList/TaskUpdate 协调任务分配。

### L3 枝干层：分镜设计（每章创作前执行）

**触发**：创作每一章之前

**产出**：章节规划 + 分镜脚本

> **📌 分级执行机制（v4.0 新增）**
> 
> | 章节类型 | 分镜字数 | 规划字数 | 适用场景 |
> |---------|---------|---------|---------|
> | **日常章** | 200字 | 150字 | 铺垫、过渡、信息交代 |
> | **普通章** | 350字 | 250字 | 大部分剧情章节 |
> | **关键章** | 500字 | 300字 | 转折点、高潮、重大事件 |
> 
> **判断规则**：
> - 章节涉及「重大冲突」「角色死亡」「情感转折」「爽点爆发」→ 关键章
> - 章节是「首次登场」「重要配角」「新场景」→ 普通章
> - 其他 → 日常章

#### S3.5 技术规格（分镜后必填）v4.2融合OLD

> **📌 阶段耦合说明（OLD-SOP核心）**：S3→S3.5→S4 为串行链路，S3.5 依赖 S3 产出，若 S3 修改需同步更新 S3.5

| 字段 | 说明 | 指标 |
|------|------|------|
| **信息密度** | 高/中/低，控制节奏 | 日常章→低，普通章→中，关键章→高 |
| **情绪曲线** | 起→承→转→合，标注峰值点 | 例：平淡(10%)→升温(30%)→爆发(50%)→收尾(10%) |
| **钩子位置** | 第几段埋入下一章钩子 | 例：第8段/共12段 |
| **伏笔回收** | 关联前文章节号 | 例：ch012伏笔→本章回收 |
| **对话占比** | 目标30%-50%，超出预警 | 超出50%→节奏拖沓警告 |

```
- 目标：

【承】(约1000字)
- 阻碍：
- 行动：

【转】(约1000字)
- 意外：
- 升级：

【合】(约500字)
- 解决：
- 结尾悬念：
```

#### 3.2 分镜脚本（含衔接模板 + 爽点强制嵌入）
```
【chXXX】分镜脚本

上一章结尾: {精确描述最后一幕}
↓ 发生了什么:{简述时间跳跃/过渡}
本章开头: {场景+人物+行动}

【过渡模板】- 必选其一
□ 模板A-时间过渡: "X天后" + 感受/铺垫
□ 模板B-场景过渡: 情绪延续 + 空间转移 + 新事件
□ 模板C-因果过渡: 上一章结果 + 连锁反应 + 本章行动

【矛盾设计】- 必做
- 本章核心冲突: {是什么}
- 主角当前困境: {处境多糟？不行动的后果？}
- 推动力: {必须行动的理由}
- 后果: {行动失败/不行动会怎样？}

【爽点强制嵌入】- 必做（关键章必填，日常章至少1项）
- 爽点类型: 打脸/逆袭/揭露/反杀/情感突破/技术展示/其他
- 爽点位置: 章开头/中段/高潮/结尾
- 爽点呈现: {具体描写方式}
- 爽点效果: {读者预期情绪反应}

【创意与记忆点检查】- 必做（v4.1 新增）
> 灵感来源：《宇宙super英雄传》ch001 成功分析 - 创意炸点+情绪锚点是第一章成功的关键

| 检查项 | 检查内容 | 不合格表现 | 合格示例 |
|--------|---------|-----------|---------|
| **开场Hook** | 开头有没有抓人点？ | "时间-地点-人物"流水账，太平淡 | 对话引入、悬念、意外、动作 |
| **创意亮点** | 这一章的核心创意是什么？ | 说不出来，纯推进剧情 | 克苏鲁→弹幕、烤红薯反差 |
| **记忆点** | 有没有让读者记住的细节？ | 看完就忘 | "太空酒吧吃烤红薯" |
| **情绪曲线** | 有没有两种以上情绪交替？ | 全程一个调 | 逗比→神秘→搞笑→悬疑 |
| **人物出场** | 新人物怎么出场？ | "XX是XX帝国的公主"念设定 | 通过行动/对话/冲突展现 |

**自检问题**：
1. 这一章的核心画面是什么？能用一句话描述出来吗？
2. 没有文字，读者能"看到"什么？
3. 对手/反派的反应是什么？
4. 普通人会怎么想这个场景？

**质量门槛**：任何一项不合格 → 退回重写

> **📌 质检执行说明（OLD-SOP核心）**：S5质量检查阶段需完成数字逻辑、常识、NPC反应、错误分级，**P0致命错误清零**后方可进入S6发布

关键衔接点:
- 角色在哪: {位置连续性}
- 剧情因果: {为什么发生}
- 主线绑定: {与核心冲突的关联}

本章结局悬念: {为下一章留钩子}
```

### 衔接模板详解

#### 模板A：时间过渡
适用场景：时间跳跃（第二天、三天后、几小时后）

```
【承接】上一章结尾 + 过渡句 + 本章开头

公式：
"【时间词】+【感受/状态】" → "【新事件/新场景】"

示例：
"战斗结束的第三天，陈默躺在床上浑身酸痛。"
"三天足够让伤口结痂，却不够让他忘记那晚的眼神。"
"手机响起，是林清雅的专属铃声。"

要点：
- 时间词要具体（"三小时"比"不久"好）
- 过渡句要有关联（感受承接上一章情绪）
- 本章开头立即有新事件
```

#### 模板B：场景过渡
适用场景：空间切换（会议室→电梯、战场→家中）

```
【承接】情绪延续 + 空间转移 + 新事件

公式：
"【情绪动作】" → "【移动过程】" → "【新事件】"

示例：
"陈默握紧拳头，指甲陷入掌心。"
"走出会议室，他直接进了电梯。"
"电梯门关上的瞬间，手机震动了。"

要点：
- 情绪不能断（带着上一章情绪离开）
- 空间转移要简洁（一句话带过）
- 新事件要立即抓住注意力
```

#### 模板C：因果过渡
适用场景：事件连续（做完一件事→做下一件）

```
【承接】上一章结果 + 连锁反应 + 本章行动

公式：
"【决定/结果】" → "【连锁反应】" → "【新行动】"

示例：
"既然已经暴露，就没必要再装下去。"
"陈默打开系统面板，开始最后的布局。"
"A级任务触发：72小时内获取微软核心代码。"

要点：
- 因果要清晰（上章→本章的逻辑）
- 连锁反应自然（不是突然冒出来的）
- 本章行动立即开始
```

### Step 9: QA审稿（含衔接检查）- 必做

**检查项目：**

- 创新度 (避免套路重复)
- 完整度 (起承转合)
- 张力检测 (冲突是否足够？主角处境够糟吗？)
- **衔接检查 (以下5项必检)**

#### QA衔接检查清单（5项必检）

| # | 检查项 | 问题表现 |
|---|--------|---------|
| 1 | **场景连续性** | 突然换地图无过渡 |
| 2 | **时间连续性** | 时间跳跃无过渡句 |
| 3 | **情绪连续性** | 情绪突然180度转弯 |
| 4 | **因果连续性** | 事件凭空出现 |
| 5 | **信息连续性** | 新人物/物品无介绍 |

#### 衔接问题严重度分级

| 等级 | 问题 | 处理 |
|------|------|------|
| 🔴 严重 | 场景/时间突然跳跃无过渡 | **必须修改** |
| 🟠 中等 | 情绪断崖式变化 | 建议修改 |
| 🟡 轻微 | 信息介绍稍显突兀 | 可以不改 |
| 🟢 通过 | 衔接自然流畅 | 通过 |

#### 衔接问题修复模板

**问题：场景跳跃**
```
修复：加入场景转移句
原文：打完架后，他回到家。
修复：打完架后浑身是血的陈默，踉跄着走出巷子。半小时后，他终于回到了家。
```

**问题：时间断裂**
```
修复：加入时间过渡句
原文：第二天，陈默去找林清雅。
修复：战斗结束后的第二天，伤口还在隐隐作痛。陈默拨通了林清雅的电话。
```

**问题：情绪断层**
```
修复：加入情绪缓冲
原文：听到这个消息，陈默很开心。他然后去吃饭了。
修复：听到这个消息，陈默愣了一下，随即笑了。笑了三秒后，他深吸一口气，压下情绪。现在不是高兴的时候，还有正事要办。
```

## 13种结尾悬念钩子 v5.0融合chinese-novelist

> 来源：chinese-novelist-skill/references/guides/hook-techniques.md

### 基础13式

| # | 类型 | 核心 | 示例 |
|---|------|------|------|
| 1 | **突然揭示** | 出乎意料改变一切的信息 | "最后一条短信来自他自己三天前发出的号码" |
| 2 | **紧急危机** | 迫在眉睫的危险+时间紧迫 | "地板开始震动，裂缝正在迅速扩大" |
| 3 | **未完成动作** | 进行中被中断，不确定结果 | "他举起枪——身后传来：别动！" |
| 4 | **身份反转** | 被揭示为不是以为的那样 | "他拔出的竟是与自己一模一样的脸" |
| 5 | **两难选择** | 必须做出不理想的选择 | "救生艇只能载两个人，她伸出手——" |
| 6 | **神秘物品** | 发现意义不明的重要线索 | "照片拍摄于昨天，角度是从她窗外" |
| 7 | **时间限制** | 明确截止时间制造紧迫 | "定时器显示03:00，拆弹包少了一根红线" |
| 8 | **承诺/威胁** | 某人做出改变预期的承诺 | "今晚午夜前，我会让所有人知道真相" |
| 9 | **离奇消失** | 不可能的行为，缺乏解释 | "转身一秒钟后，戴着手铐的囚犯不见了" |
| 10 | **言外之意** | 暗示更深层信息的正常话语 | "恭喜你通过面试，和你姐姐一样优秀" |
| 11 | **意象钩子** | 反复铺垫的意象在结尾微妙变化 | "她终于吹灭了蜡烛。黑暗中，有什么东西在移动" |
| 12 | **回声钩子** | 结尾呼应开篇但有关键细节变化 | 开篇"一切都很简单"→结尾"终于明白没什么是简单的" |
| 13 | **留白钩子** | 只写角色对悬念的反应，不写事件 | "他看完信，脸上表情变了几变，很久没有说话" |

### 悬念强度等级

| 等级 | 类型 | 读者反应 | 适用 |
|:---:|------|---------|------|
| 1 | 好奇悬念 | "这很有趣" | 中间章 |
| 2 | 关切悬念 | "接下来会发生什么" | 中间章 |
| 3 | 迫切悬念 | "他必须马上行动" | 高潮章 |
| 4 | 生存悬念 | "他会活下去吗" | 高潮/结局前 |
| 5 | 终极悬念 | "一切到底是什么意思" | 全书结尾 |

> **递进原则**：悬念强度应总体上升，但可波动。20章小说通常2-3个高强度锚点，其余用中低强度衔接。过度使用高强度反而让读者疲劳。

### 跨章悬念弧设计

| 弧线 | 跨度 | 功能 |
|------|------|------|
| **短弧** | 2-3章 | 即时满足，维持节奏 |
| **中弧** | 5-8章 | 渐进揭露，形成波峰 |
| **长弧** | 全书 | 核心谜题，贯穿始终 |

## 10种强力开头技巧 v5.0融合chinese-novelist

> 来源：chinese-novelist-skill/references/guides/chapter-guide.md

### 前20%决定生死
读者在前20%决定是否继续阅读。如果开头不够吸引人，读者会放弃，无论后面多么精彩。

### 开头致命错误（绝对避免）

| 错误 | 示例 | 为什么致命 |
|------|------|-----------|
| 天气描写 | "那天天气晴朗……" | 与故事无关 |
| 日常流程 | "李明醒来，刷牙洗脸……" | 无冲突 |
| 回顾上章 | "上一章我们说到……" | 浪费时间 |
| 缓慢铺垫 | "先介绍这个城市背景……" | 信息倾倒 |
| 平淡对话 | "你好，你好吗？" | 无张力 |
| 过度解释 | "这是因为，所以……" | 讲述而非展示 |

### 10种开头技巧

| # | 技巧 | 核心 | 示例 |
|---|------|------|------|
| 1 | **行动中开场** | 从冲突高潮点开始 | "子弹擦过他的耳边，李明没有回头，翻滚躲到沙发后面" |
| 2 | **反常情境** | 不符合常理的场景 | "死人坐在办公桌前，正在写一份报告" |
| 3 | **震撼对话** | 一句惊人的对话 | "我把孩子卖了。"妻子平静地说 |
| 4 | **倒计时开场** | 时间压力制造紧迫 | "还有三分钟，炸弹就会爆炸。而拆弹包里少了一根红线" |
| 5 | **重大发现** | 发现关键线索或真相 | "死因不是意外，是精心策划的谋杀。嫌疑人只有一个——他自己" |
| 6 | **危机时刻** | 面临最大危机 | "门被踹开的瞬间，她知道藏不住了" |
| 7 | **谜团浮现** | 无法解释的现象 | "每天凌晨三点零七分，她的手机会收到一条空白短信。发送者号码是她自己的" |
| 8 | **背叛开场** | 背叛或信任崩塌 | "枪口对准他的后背。'对不起，'他信任了十年的搭档说" |
| 9 | **重大选择** | 艰难决定开始 | "救生艇只能载一个人。她的丈夫和女儿都在水里挣扎。她伸出手——" |
| 10 | **结局预告** | 从未来关键时刻开始 | "三天后，所有人都会后悔今天的决定。但此刻，每个人都在微笑" |

### 打破读者预期

| 技术 | 操作要点 |
|------|---------|
| **预期反转** | 铺垫2-3段让读者确信自己的判断；反转不能凭空出现；反转后要比原预期更有趣 |
| **信息差利用** | 读者知道角色不知道的事（制造焦虑），或角色知道读者不知道的事（制造悬念）；持续不超过2-3章 |
| **复杂动机** | 没有纯粹的善恶；揭示时机在读者已形成判断之后；最好效果：读者不知道该站在谁那一边 |
| **节外生枝** | 在看似已解决的问题上引入新变量；引入时机在读者刚刚放松警惕的瞬间；新变量要比原来的问题更大 |

## 去AI味深度润色 v5.0融合chinese-novelist

> 来源：chinese-novelist-skill/references/flows/phase3-writing.md

### 深度润色（去除AI味）检查清单

重点检查并修改：

1. **去除过度修饰的形容词** — 删除"璀璨"、"瑰丽"、"绚丽多彩"等AI常用词堆砌
2. **减少抽象陈述** — 把"心中涌起复杂的情感"改为具体动作/对话
3. **打破四字格律** — 避免"心潮澎湃、热血沸腾"等陈词滥调
4. **增加口语化表达** — 人物对话要有个性，不要都是播音腔
5. **优化节奏感** — 长句短句交替，不要全是同等长度的句子
6. **细节具象化** — 用具体细节替代笼统描述
7. **动作代替形容** — "他老了"不如"他扶着墙站起来"

### 章节字数标准

| 章节类型 | 字数要求 | 不足时 |
|----------|---------|--------|
| 日常章 | 2000-3000字 | 使用 content-expansion 扩充技巧 |
| 普通章 | 3000-4000字 | 使用 content-expansion 扩充技巧 |
| 关键章 | 4000-6000字 | 必须达标，高潮需要充分展开 |

> 字数检查必须使用脚本：`python scripts/check_chapter_wordcount.py <章节文件路径>`

## 自动校验与3轮重写 v5.0融合chinese-novelist

### 校验流程

1. **字数检查** — 每章必须达到目标字数
2. **连贯性检查** — 人物一致性、情节连贯、节奏控制
3. **不合格章节自动重写** — 最多3轮

### 重写规则

| 轮次 | 条件 | 操作 |
|------|------|------|
| 第1次 | 字数<3000 或 连贯性差 | 记录问题，重新生成 |
| 第2次 | 仍不合格 | 分析失败原因，调整策略 |
| 第3次 | 仍不合格 | 记录错误到 QA 报告，跳过本章 |
